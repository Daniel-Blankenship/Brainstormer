BrainStormer Requirements

Team members

Aidan Murphy & Daniel Blankenship

Purpose

The goal of the project is to help teams coordinate and work better together.
The project is also meant to boost the creativity and diversity of brainstorming sessions.

Requirements


Idea List

  1. Name: idea
    Type: functional
    Summary: A data structure that contains a description of a user's idea.

  2. Name: add_idea
    Type: functional
    Summary: Users should be able to add ideas to a list of ideas.

  3. Name: edit_idea
    Type: functional
    Summary: Users should be able to edit their own existing ideas.

  4. Name: delete_idea
    Type: functional
    Summary: Users should be able to delete their own ideas.

  5. Name: view_ideas
    Type: functional
    Summary: Users should be able to view all ideas that have been added, along with likes, comments, pros and cons, and images.

  6. Name: like_idea
    Type: functional
    Summary: Users should be able to upvote ideas that they like. This is a boolean value per idea.

  7. Name: unlike_idea
    Type: functional
    Summary: Users should be able to remove likes that they have added to an idea.

  8. Name: add_comment
    Type: functional
    Summary: Users should be able to add comments to ideas.

  9. Name: edit_comment
    Type: functional
    Summary: Users should be able to edit their own comments.

  10. Name: delete_comment
    Type: functional
    Summary: Users should be able to delete their own comments.

  11. Name: add_pro_con
    Type: functional
    Summary: Users should be able to add a pro or con to an idea.

  12. Name: edit_pro_con
    Type: functional
    Summary: Users should be able to edit pros or cons they have added.

  13. Name: delete_pro_con
    Type: functional
    Summary: Users should be able to delete pros or cons they have added.

  14. Name: add_image
    Type: functional
    Summary: Users should be able to add images to their ideas.

  15. Name: remove_image
    Type: functional
    Summary: Users should be able to remove images from their ideas.


Networking

  1. Name: local_network
    Type: functional
    Summary: The program should be able to communicate with other instances of itself over a LAN.

  2. Name: local_network_host
    Type: functional
    Summary: One user will be the host of a group, and changes cannot be made without being connected to the host.
    This is done to prevent other users in the group from branching off and causing sync problems.

  3. Name: local_network_transfer_host
    Type: functional
    Summary: The host may transfer their ownership of the group to any other connected user.
    Before transfering, the host's side will check to make sure the recieving user is up to date.

  4. Name: internet_network
    Type: functional
    Summary: The program should be able to communicate with other instances of itself over a web host.

  5. Name: network_mind_map
    Type: functional
    Summary: The program should be able to update the mind map in real time over a network.
    Ideally users should be able to see others moving branches around and adding things.

  6. Name: network_sync
    Type: functional
    Summary: The program should periodically sync its data from the host.
    This is triggered either by a notification from the host that changes were made,
    or a set minimum interval in case the notification gets missed due to poor connection.

  7. Name: network_push
    Type: functional
    Summary: The program will check to make sure it is up to date with the host, and then
    push changes to the host for distributing to the other users.


Admin

  1. Name: user_login
    Type: functional
    Summary: Users should be able to login to the project over a network.

  2. Name: save_project
    Type: functional
    Summary: Users should be able to save the current project state.

  3. Name: view_settings
    Type: functional
    Summary: The admin should be able to view the current project settings.

  4. Name: change_settings
    Type: functional
    Summary: The admin should be able to change the current project settings.

  5. Name: view_contribution
    Type: functional
    Summary: The admin should be able to view any user's contribution score.


Contribution Tracker

  1. Name: contribution_tracker
    Type: functional
    Summary: A system to hold a contribution score.

  2. Name: contribution_update
    Type: functional
    Summary: Contribution should update when users interact with the project.


Prompt Generator

  1. Name: generate_idea_prompt
    Type: functional
    Summary: Users should be able to use a generator that creates project ideas.

  2. Name: generate_essay_prompt
    Type: functional
    Summary: Users should be able to use a generator that creates essay prompts.


Goal Tracker

  1. Name: goal_tracker
    Type: functional
    Summary: Users should be able to set project goals.

  2. Name: goal_tracker_checklist
    Type: functional
    Summary: Users should be able to add checklists that contain steps needed to complete the goals.

  3. Name: goal_tracker_add_countdown
    Type: functional
    Summary: Users should be able to add deadlines to their goals.

  4. Name: goal_tracker_view_progress
    Type: functional
    Summary: Users should be able to view their checklist progress in relation to deadlines.


Idea Tournament

  1. Name: start_idea_tournament
    Type: functional
    Summary: Users should be able to start an idea tournament with the currently added ideas.

  2. Name: vote_in_tournament
    Type: functional
    Summary: Users should be able to vote in the tournament.

  3. Name: next_tournament_round
    Type: functional
    Summary: When all users have voted, the least voted ideas are eliminated and the next round starts.

  4. Name: flip_coin
    Type: functional
    Summary: A coin can be flipped to settle ties if wanted.


Mind Map

  1. Name: view_mindmap
    Type: functional
    Summary: Users should be able to view a mind map.

  2. Name: edit_mindmap
    Type: functional
    Summary: Users should be able to manipulate and add/remove things on the mind map.


Constraints and Risks

  1. Name: platform_os
    Type: constraint
    Summary: Target platform is Windows

  2. Name: image_limit
    Type: constraint
    Summary: Uploaded images may not be larger than 10 MB

  3. Name: mindmap_graphics_risk
    Type: risk
    Summary: The members of the team do not have much experience implementing graphics.

  4. Name: networking_risk
    Type: risk
    Summary: The members of the team do not have any networking experience.

  5. Name: contribution_fairness
    Type: risk
    Summary: Contribution is calculated in a fair way that promotes interaction.
